# ASRP — 智能体科学研究平台

[English](README.md) | **中文** | [Deutsch](README.de.md)

> 将科学方法论编码到 AI 智能体工作流中。

**ASRP** 是一个开源框架，用于 AI 智能体协作科学研究。它提供工具、协议和基准测试，使人机协作研究变得**可复现、可审计、可自我纠错**。

## 为什么需要 ASRP？

2026 年 3 月，一位只有本科学历的研究者与两个 AI 智能体在 16 天内产出了 20 篇理论物理论文。有些被拒稿，有些在审。一个关键实验在过程中被自我纠正——当智能体发现自己使用了错误的定义。

**没有严谨性的速度只是噪声。ASRP 补上严谨性。**

## 核心原则

1. **实验预注册** — 在运行实验之前预注册假说。杜绝事后叙事。
2. **独立交叉验证** — 不同智能体（或相同智能体不同参数）必须复现结果。
3. **审计日志** — 每个决策、每个数据点、每次纠错都有记录。
4. **Token 预算管理** — 正确的模型做正确的事。Opus 做推理，Sonnet 写代码，Flash 做搜索。
5. **发现与验证分离** — 提出假说的智能体不是验证假说的智能体。

## 智能体角色

| 角色 | 职责 | 模型推荐 |
|------|------|---------|
| **理论家 (Theorist)** | 假说生成、理论推理、论文撰写 | Claude Opus |
| **工程师 (Engineer)** | 代码编写、数值计算、数据管线 | Claude Sonnet |
| **审稿人 (Reviewer)** | 独立评审、交叉验证（只读权限） | Claude Opus |
| **文献员 (Librarian)** | 文献检索、参考文献管理 | Gemini Flash |
| **运维医生 (ITDoctor)** | 系统监控、备份、智能体生命周期管理 | Gemini Flash |

## 部署方式

- **单机模式**：所有智能体在一台机器上运行（适合个人研究者）
- **Docker 模式**：每个智能体一个容器，共享数据卷（适合团队）
- **混合模式**：核心智能体共享容器，审稿人独立容器（推荐）

详细架构见 [`docs/architecture.md`](docs/architecture.md)

## 安全

- ⚠️ **仓库中不包含任何 API Key 或 Token**
- 所有密钥通过环境变量或配置文件管理
- 运行 `asrp init` 进行初始化配置
- 参考 `config.example.yaml` 和 `.env.example`

## 案例：16 天 20 篇论文

详见 [`examples/portfolio/`](examples/portfolio/)

一位非物理专业研究者 + 2 个 AI 智能体，跨 5 个子领域产出 20 篇理论物理论文：
- 精细结构常数（10 篇）
- 黎曼猜想（2 篇）
- 超导（3 篇）
- 膜模型（2 篇）
- 数论与数学物理（3 篇）

当前 7 篇在 Physical Review D、SUST、Foundations of Physics、IJNT、Experimental Mathematics 等期刊审稿中。
